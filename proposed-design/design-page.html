🔌 Provider Packs Design LLM Provider Decoupling — v3 (post architecture review)

Problem

Model IDs and provider config are hardcoded across 5+ files per pack. Each agent pack independently handles provider selection, making it impossible to add new providers without modifying every pack.

Goal

Decouple LLM provider selection from agent pack selection. User picks three things:

🧩 Agent Pack

openclaw, hermes, pi, claude-code, ironclaw, nemoclaw, kiro-cli

🛡️ Permission Profile

builder, account_assistant, personal_assistant

🧠 LLM Provider

bedrock, anthropic-api, openai-api, openrouter, litellm

Deploy Wizard UX

The web-based deploy wizard is the primary user-facing entry point. It must reflect the new three-choice model and guide users through provider selection with clear defaults and validation.

Existing features not yet in provider-packs design (will be added in implementation):
  • CFN/TF params: EnvironmentName, OpenClawGatewayPort, LokiWatermark, RepoBranch, VpcCidr, PublicSubnetCidr, SSHAllowedCidr, KeyPairName, ExistingVpcId, ExistingSubnetId
  • Security toggles: EnableSecurityHub, EnableGuardDuty, EnableInspector, EnableAccessAnalyzer, EnableConfigRecorder (all default true)
  • Bedrock onboarding: RequestQuotaIncreases, EnableBedrockForm
  • Instance families: c7g (compute-optimized) — current design shows t4g + m7g only
  • Pack-specific params: Claude Code --haiku-model, bedrockify --embed-model, NemoClaw --sandbox-name / --telegram-token / --allowed-chat-ids
  • NemoClaw restrictions: compatible_profiles: [personal_assistant] only, requires Docker, t4g.xlarge minimum
  • Kiro CLI: Own cloud inference (no Bedrock/provider needed), requires post-install interactive login (kiro-cli login --use-device-flow)
  • bootstrap.sh params: --gw-port, --model-mode, --bedrockify-port, --hermes-model (all forwarded to pack config JSON)

These are existing install.sh / template.yaml / variables.tf features that the provider-packs redesign must preserve or migrate. The new --provider flag replaces --model-mode, but all other params remain.
